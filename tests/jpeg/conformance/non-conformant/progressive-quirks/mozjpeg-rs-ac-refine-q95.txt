# mozjpeg-rs AC Refinement Progressive JPEG (Q95)

## Description

Progressive JPEG encoded by mozjpeg-rs v0.3.0 using the `ProgressiveBalanced` preset
at quality 95. Uses the JCP_MAX_COMPRESSION 9-scan script with AC successive
approximation refinement (Ah > 0 scans).

Image: 167x184 pixels, 4:4:4 subsampling, gradient + checkerboard pattern.

## What's Non-Conformant

The AC refinement scan bitstream appears to have an edge case that strict Huffman
decoders reject. The exact cause is under investigation, but likely involves:
- AC refinement scans with many coefficients to refine (high quality = low quantization)
- Specific coefficient patterns from gradient images
- Possible EOBRUN or correction bit handling issue

## Decoder Behavior

| Decoder | Result |
|---------|--------|
| mozjpeg / libjpeg-turbo | Decodes successfully (lenient) |
| jpeg-decoder (Rust) | "failed to decode huffman code" |
| zune-jpeg (Rust) | "Bad Huffman Code 0xFFFF, corrupt JPEG" |
| image-rs | Fails (uses jpeg-decoder internally) |

## Expected Strict Behavior

Reject with Huffman decoding error.

## Expected Lenient Behavior

Decode successfully with correct image output.
C mozjpeg emits warning "Corrupt JPEG data: bad Huffman code" but produces correct pixels.

## Workaround

Use `Preset::ProgressiveSmallest` or `.optimize_scans(true)` which avoids AC
refinement scans at high quality levels by using the scan optimizer.

## Source

Generated by mozjpeg-rs v0.3.0 for decoder conformance testing.
See: https://github.com/imazen/mozjpeg-rs

## Reproduction

```rust
use mozjpeg_rs::{Encoder, Preset, Subsampling};

let width = 167u32;
let height = 184u32;
let mut pixels = vec![0u8; (width * height * 3) as usize];
for y in 0..height as usize {
    for x in 0..width as usize {
        let idx = (y * width as usize + x) * 3;
        pixels[idx] = (x * 255 / width as usize) as u8;
        pixels[idx + 1] = (y * 255 / height as usize) as u8;
        pixels[idx + 2] = if (x / 8 + y / 8) % 2 == 0 { 200 } else { 50 };
    }
}

let jpeg = Encoder::new(Preset::ProgressiveBalanced)
    .quality(95)
    .subsampling(Subsampling::S444)
    .encode_rgb(&pixels, width, height)?;
```
